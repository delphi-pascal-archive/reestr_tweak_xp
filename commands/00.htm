Очистка файла подкачки перед перезагрузкой системы

Очищает файл pagefile.sys перед перезагрузкой системы. В данном файле может храниться конфиденциальная информация. Если данная опция включена, возможно значительное увеличение времени выключения компьютера.

HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management

DWORD ClearPageFileAtShutdown, значение 1 или 0

Наверх
Отключить встроенный отладчик Dr. Watson

Отключает встроенного отладчика Windows XP (программа обнаружения ошибок). Если текущим отладчиком не является Dr. Watson, он будет также отключен. Если опция выключена, то при ошибке в программе, будет выводиться соответствующее сообщение об ошибке с кнопками ОК и Отмена, при нажатии ОК - приложение завершается, а при нажатии на отмену запускается отладчик.

Примечание: можно отключить вывод окошка совсем см. опцию Сообщения об ошибках

HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\AeDebug

DWORD Auto, значение 1 или 0

Не производить запись последнего доступа к файлам (только NTFS)

Ускоряет доступ к каталогам с большим количеством файлов. Данная опция работает только с дисками с файловой системой NTFS. На работу FAT32 никак не влияет. Чтобы узнать файловую систему диска, щелкните правой кнопкой на имени диска в Моем компьютере и выберите пункт меню свойства.

HKLM\SYSTEM\CurrentControlSet\Control\FileSystem

DWORD NtfsDisableLastAccessUpdate, значение 1 или 0

Наверх
Отключить System Files Protection (SFC)

Отключает блокировку системных файлов Windows XP, будьте осторожны с этой опцией, т.к. системные фалы могут быть заменены старыми версиями.

HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon

DWORD SfcDisable, параметр может принимать различные значения, XP Tweaker для отключения SFC ставит значение hex:ffffff9d (4294967197) или 0 для включения

Начиная с версии 1.30, XP Tweaker предлагает внести изменения в системную библиотеку sfc_os.dll. Предварительно программа делает копию файла sfc_os.xpt, проверяет версию и только после этого изменяет файл. Это полностью отключит SFC. При отмене опции программа восстановит файл sfc_os.xpt, если он существует.

Примечание: Если после применения опции был установлен SP1, то необходимо удалить файл sfc_os.xpt из системной папки и после этого заново применить опцию.

Включить поддержку UDMA-66 на чипсетах Intel *

Необходимо перезагрузить компьютер, чтобы изменения вступили в силу.

HKLM\System\CurrentControlSet\Control\Class\{4D36E96A-E325-11CE-BFC1-08002BE10318}\0000

DWORD EnableUDMA66, значение 1 или 0

Примечание: прежде чем применить опцию, убедитесь, что ваше устройство поддерживает UDMA-66 и используется 80-pin кабель.
Данная возможность не должна использоваться для UDMA-100, необходимо установить драйвера контролера IDE.

Автоматически выгружать не используемые библиотеки

Разрешает Windows XP выгружать не используемые библиотеки, тем самым, освобождая память. Если наблюдается нестабильная работа системы при включенной опции, отключите её.

HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer

DWORD AlwaysUnloadDLL, значение 1 - выгружать библиотеки, значение 0 - не выгружать

Примечание: включение данной опции может привести к невозможности запуска XP Tweaker, в этом случае исправить значение можно только при помощи редактора реестра!

Отключить слежение Windows XP за пользователем

Windows XP постоянно записывает действия пользователя, какие программы он запускал, какие открывал документы и т.д. В целях безопасности данную опцию можно отключить.

HKCU\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer

DWORD NoInstrumentation, значение 1 - отключить слежение, значение 0 - включить

Примечание: НЕ БУДЕТ СОХРАНЯТЬСЯ список часто вызываемых программ (в меню Пуск), недавние документы и пр.

Запускать 16-битные программы в отдельных процессах

Запускает приложения для MS-DOS и Windows 3.1 в отдельных процессах. При крахе одного процесса, работа других не нарушится. Включение данной опции потребует больше памяти под 16-битные программы.

HKLM\SYSTEM\CurrentControlSet\Control\WOW

STRING DefaultSeparateVDM, значение "yes" - запускать программы в отдельных процессах, другие игнорируются

 

Ссылки по теме:

Не отсылать в Microsoft отчеты об ошибках

С помощью данной опции можно отключить отсылку сообщений об ошибках в Microsoft. Данную опцию, при необходимости, можно более подробно настроить в Свойствах системы, вкладка Дополнительно, кнопка отчет об ошибках.

HKLM\SOFTWARE\Microsoft\PCHealth\ErrorReporting

DWORD DoReport, 0 - Отключить отчет об ошибках, 1 - включить
DWORD ShowUI, 0 - Отключить сообщения о критических ошибках, 1 - включить сообщения о критических ошибках, 2 - включить сообщения обо всех ошибках.

Если данная опция включена, XP Tweaker присваивает параметрам DoReport и ShowUI значение 0, иначе обоим присваивает значение 1. Вы также можете настроить некоторые другие опции (они будут работать, только если DoReport = 1 и ShowUI = 3)

DWORD IncludeKernelFaults - Отчет об ошибках Windows XP (ошибки ядра), может принимать значения 1 или 0
DWORD IncludeMicrosoftApps - Отчет об ошибках программ от Майкрософт, может принимать значения 1 или 0
DWORD IncludeWindowsApps - Отчет об ошибках компонентов Windows XP, может принимать значения 1 или 0

DWORD AllOrNone - Дает возможность запрета отсылки сообщений об ошибках определенных приложений, значение 1 - отсылаются все, значение 0 - только выбранные приложения. Для того чтобы выбрать приложение, его необходимо добавить в
HKLM\SOFTWARE\Microsoft\PCHealth\ErrorReporting\InclusionList, а чтобы исключить в
HKLM\SOFTWARE\Microsoft\PCHealth\ErrorReporting\ExclusionList.

В данных разделах необходимо создать DWORD параметр с названием программы и значением 1, например:

HKLM\SOFTWARE\Microsoft\PCHealth\ErrorReporting\ExclusionList
"programm.exe"=dword:00000001

исключает отсылку сообщений об ошибках программы programm.exe

Оптимизировать системные файлы во время загрузки (boot defrag)

Дефрагментация файлов, необходимых для старта операционной системы. Выключение этой функции позволит на некоторое время уменьшить время загрузки, но со временем она будет становиться все медленнее.

HKLM\SOFTWARE\Microsoft\Dfrg\BootOptimizeFunction

STRING Enable, значение "Y" - оптимизация включена, "N" - выключена

Принудительный вызов "стоп-ошибки" при нажатии Ctrl+Scroll Lock *

Принудительный вызов "синего экрана смерти" вручную. Данную возможность можно использовать при зависании компьютера вместо нажатия reset.

Необходимо перезагрузить компьютер, чтобы изменения вступили в силу. После этого для вызова стоп-ошибки нужно удерживая клавишу Ctrl два раза нажать на Scroll Lock.

HKLM\SYSTEM\CurrentControlSet\Services\i8042prt\Parameters

DWORD CrashOnCtrlScroll, значение 1 - принудительный вызов ошибки вручную включен

Быстрое редактирование в командной строке

Позволяет быстро копировать/вставлять текст в консоли, используя кнопки мыши. Левая кнопка используется для выделения текста в консоли, правая для копирования (если в консоли имеется выделенный текст) или для вставки текста. Чтобы настройка вступила в силу, необходимо перезапустить консоль.

HKCU\Console

DWORD QuickEdit, значение 1 - быстрое редактирование включено, 0 - выключено.

Примечание: если опция включена, контекстное меню будет не доступно

Выводить окно для ввода причины завершения работы

Windows Server 2003 не дает завершить работу компьютера без ввода причины завершения. С помощью данной опции можно отключить этот запрос. В Windows XP этот режим по умолчанию выключен.

HKLM\Software\Policies\Microsoft\Windows NT\Reliability

DWORD ShutdownReasonUI, значение 1 - запрос причины завершения включен, 0 - выключен.

Примечание: если параметр отсутствует, то Windows XP считает его равным 0, а Windows Server 2003 равным 1

Переписать консоль восстановления на жесткий диск

Консоль восстановления Windows находится на CD-ROM с Windows XP, но имеется возможность переписать её на жесткий диск так, что при загрузке вы сможете выбрать: загружать XP или консоль восстановления. Перед тем как щелкнуть по ссылке вставьте компакт-диск с дистрибутивом Windows XP.

XP Tweaker выполнить команду: %путь к дистрибутиву%\i386\winnt32.exe /cmdcons

После установки консоли, на системном диске появится папка cmdcons размером 7-8 Мб, а в boot.ini появиться новая строка

...\CMDCONS\BOOTSECT.DAT="Microsoft Windows XP Recovery Console" /cmdcons

Выполните Пуск->Выполнить->msconfig перейдите на вкладку boot.ini и убедитесь, что появился новый пункт меню.

Автоматические обновления Windows

Управление автоматическим обновлением Windows XP. Также данную опцию можно настроить следующим способом: Панель управления - Система - Автоматическое обновление.

HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update

DWORD AUOptions
DWORD AUState

Windows XP использует следующие значения:

Отключить автоматические обновления:

AUOptions = 1
AUState = 7

Уведомлять о возможности загруки обновлений

AUOptions = 2
AUState = 2

Загружать обновления, затем уведомлять о готовности к установке

AUOptions = 3 и другие значения
AUState = 2

Windows Prefetcher сервис

В Windows XP есть специальный Prefetcher Service, его задача уменьшить время запуска приложений и время загрузки системы. Файлы, используемые этим сервисом, находятся в папке %systemroot%\prefetch. C помощью данной опции Вы можете управлять данным сервисом.

HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management\PrefetchParameters

DWORD EnablePrefetcher, параметр может принимать следующие значения:

0 - отключить сервис

1 - упреждающая выборка приложений

2 - упреждающая выборка приложений, запускаемых во время загрузки системы (используется Windows Server 2003 по умолчанию)

3 - включены обе упреждающие выборки (пп. 1 и 2) используется Windows XP по умолчанию

Перезагрузите компьютер, чтобы изменения вступили в силу.

Настройка кэша второго уровня

Опция задает значение кэша процессора второго уровня (L2 cache). Это значение используется, в случае если Windows XP не сможет определить размер кэша и позволяет полностью использовать ресурсы процессора.

HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management

DWORD SecondLevelDataCache, значение - размер кэша второго уровня процессора в килобайтах, если значение параметра равно 0, используется кэш равный 256 килобайт

Примечание: размер кэша вашего процессора можно узнать из документации к нему.

Синхронизация времени

Интервал синхронизации системных часов компьютера с time-сервером

Синхронизация времени вашего компьютера с сервером в Интернете. По умолчанию Windows XP использует интервал в 7 дней. Чтобы настроить сервер, с которым Windows XP будет сверять время, выберите в Панели управления - Дата и время, затем перейдите на вкладку Время Интернета.

HKLM\SYSTEM\ControlSet001\Services\W32Time\TimeProviders\NtpClient

Примечание: Значение интервала вступит в силу только после синхронизации времени Windows или вручную.

Работа Windows с zip-архивами

Отключить поддержку Windows XP zip-архивов

По умолчанию Windows XP считает zip-файлы папками - это удобно, если у вас быстрый компьютер. На медленных компьютерах можно отключить эту возможность. Также Windows не будет искать файлы в архивах, что значительно ускорит поиск.

Для отключения поддержки XP Tweaker исполняет команду:

regsvr32 /u /s zipfldr.dll, а для включения regsvr32 /s zipfldr.dll

Параметр /u отключает поддержку, /s не выдаются сообщения. Для определения включена поддержка или нет, XP Tweaker проверяет, существует ли параметр HKEY_CLASSES_ROOT\.zip\CompressedFolder\ShellNew BINARY Data. Если существует - поддержка включена, иначе выставляется галочка, что означает - поддержка zip-архивов отключена.

Примечание: опция не доступна для Windows Server 2003

Отказ системы

Автоматически перезагрузить компьютер

Майкрософт всегда думает о пользователях, не прошёл мимо и факт зависания компьютера. Теперь вам не надо тянуться к кнопке Reset, компьютер перезагрузиться самостоятельно, если произойдет непоправимая ошибка. Данную опцию можно настроить в Свойствах системы, вкладка Дополнительно, раздел Загрузка и восстановление.

HKLM\SYSTEM\CurrentControlSet\Control\CrashControl

DWORD AutoReboot, значение 1 - перезагружать компьютер, 0 - не перезагружать

Наверх
Записать событие в системный журнал

Производит запись в журнал об ошибке системы, если компьютером пользуется несколько человек и вам интересно, когда произошло зависание компьютера, включите данную опцию, иначе имеет смысл её отключить.

HKLM\SYSTEM\CurrentControlSet\Control\CrashControl

DWORD LogEvent, значение 1 - вести запись журнала, 0 - не вести

Примечание: опция не используется для Windows Server 2003

Отправить административное оповещение

Оповещение администраторов системы при возникновении ошибки.

HKLM\SYSTEM\CurrentControlSet\Control\CrashControl

DWORD SendAlert, значение 1 - отправлять сообщение администраторам, 0 - не отправлять

Наверх
Запись отладочной информации

Данную опцию следует включать, если вы гений в двоичных кодах :) Для всех остальных рекомендуется отключить данную опцию, т.к. при непоправимой ошибке система может довольно долго записывать отладочную информацию и её размер может быть равен размеру оперативной памяти, в зависимости от выбранных деталей отчета.

HKLM\SYSTEM\CurrentControlSet\Control\CrashControl

DWORD CrashDumpEnabled, значение 0 - запись не ведется, 1 - Полный дамп памяти (создание файла, равного размеру оперативной памяти) , 2 - Дамп памяти ядра, 3 - Малый дамп памяти (64КБ)

Параметры работы с памятью

Не использовать файл подкачки для хранения ядра системы

По умолчанию WinXP выгружает ядро и системные драйвера в файл подкачки, если они не используются. С помощью данной опции можно указать WinXP хранить ядро и системные драйвера всегда в памяти.

HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management

DWORD DisablePagingExecutive, 1 - не использовать файл подкачки для хранения ядра системы, 0 - использовать (по умолчанию)

Оптимизировать работу системного кэша *

Используйте данную опцию, если у вас 512 МБ памяти или больше. Данная опция увеличит производительность Windows XP. Рекомендуется использовать данную опция, если компьютер используется в качестве сервера или работа ведется в программах, для которых требуется кэш большого размера.

HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management

DWORD LargeSystemCache, значение 1 - использовать, 0 - не использовать

Примечание: в Windows Server 2003 по умолчанию опция включена.

 

Ссылки по теме:

Windows 2000 Resourse Kits (Memory Management, LargeSystemCache)

Оптимизировать кэш диска

Примечание:

Эта опция только для Windows NT и Windows 2000 без сервис пака.

 

Данная опция увеличит число страниц, которые система будет читать или писать на жесткий диск за один раз. Исходя из этого, должна вырасти производительность.

HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management

DWORD IOPageLockLimit, значение параметра задаётся в шестнадцатеричной системе от hex:1 до hex:FFFFFFFF байт. Если параметр не существует или равен 0 (по умолчанию), то используется кэш равный 512 Кб. Рекомендуются следующие значения:

Объём RAM компьютера

Рекомендуемое значение параметра IOPageLockLimit
128 МБ и более 16384 КБ (16 МБ)
160 МБ и более 32768 КБ (32 МБ)
256 МБ и более 65536 КБ (64 МБ)
512 МБ и более 131072 КБ (128 МБ)

Объём RAM компьютера

Максимальное значение параметра
Менее 64 МБ Объём RAM минус 7 МБ
64 МБ - 500 МБ Объём RAM минус 16 МБ
512 Мб и более Объём RAM минус 64 МБ

Консоль

Автоматически выполнять команды, при запуске консоли

Опция позволяет указать командный файл, который будет запускаться каждый раз при запуске консоли.

HKEY_CURRENT_USER\SOFTWARE\Microsoft\Command Processor

STRING AutoRun, значение - путь к командном файлу

Примечание: путь к файлу необходимо указывать по правилу 8.3 (например, "C:\DOCUME~1\dmc\РАБОЧИ~1\Autocm.cmd")
