Максимальный размер передаваемого блока данных (Maximum Transmission Unit)

Определяет самое большое количество данных, которые могут быть переданы по сети в одном физическом фрейме. Если пакет IP большего размера чем MTU, то произойдет фрагментация, что может увеличить время, которое требуется, чтобы послать единственный пакет.

По-умолчанию Windows XP использует значение 1500.

HKLM\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters

DWORD MTU, значение - размер блока в байтах

Рекомендуемые значения:

Для модема: 576

Для DSL, кабельных модемов, T1 и выше: 1500

Максимальный размер окна для приёма TCP пакетов (RWIN)

Рекомендуется изменять опцию только опытным пользователям. Под NT-системами значение RWIN является динамическим.

Количество байт, которые отправитель может передать, не получая подтверждения.

HKLM\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters

DWORD TCPWindowSize, значение - размер в байтах

XP Tweaker расчитывает значение следующим образом:

(Размер MTU - 40 байт заголовка TCP/IP) * множитель

Множитель может принимать значения от 2 - 10.

Автоматически определять MTU

Автоматически определять максимальный размер передаваемого блока данных.

HKLM\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters

DWORD EnablePMTUDiscovery, значение 1 - определять MTU автоматически
Включить поддержку TCP окон больше 64Кб

Рекомендуется для сетей с большой пропускной способностью.

HKLM\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters

DWORD Tcp1323Opts, значение 1 - поддержка включена
Максимальное время жизни пакета IP (Time to Live)

TTL определяет максимальное время, которое пакет IP может прожить в сети, не достигая адресата. Другими словами: использование этой опции определяет, как долго компьютер будет ждать, прежде чем решит, что запрашиваемый вебсайт не найден. Для плохих соединений имеет смысл увеличить данное значение.

HKLM\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters

DWORD DefaultTTL, значение - время жизни пакета.
Выборочная передача поврежденных данных

Данная опция включает поддержку Selective Acknowledgement (SACK). Если пакет или ряд пакетов TCP потеряны, то получатель может сообщить отправителю, какие данные были получены, а какие нет.

HKLM\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters

DWORD SackOpts, значение 1 - поддержка выборочной передачи включена
